# Repo Instructions

This repo has multiple subprojects. Read only the instructions for the area you are working in.

## Routing

- `career-ops/CLAUDE.md` and `career-ops/AGENTS.md` govern Career-Ops tasks.
- `signalrank/backend/README.md` governs backend work.
- `signalrank/frontend/AGENTS.md` points back to the frontend rules.

## Context Rules

- Do not preload every Markdown file in the repo.
- Read the smallest set of files needed for the task.
- Prefer the current mode file, profile file, and the relevant code or data files.
- Keep user-specific changes in the designated profile files for the subproject.

## MCP Rules

- Treat MCP servers as opt-in, not default.
- Use an MCP server only when the task explicitly needs that external integration.
- Prefer local files, scripts, and repo code over MCP tools when either will do.
